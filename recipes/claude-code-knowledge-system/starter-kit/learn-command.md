# /learn (복붙용 generic 버전)

> 이 파일을 `.claude/commands/learn.md`로 복사하면 `/learn`으로 불린다.
> 원본(실서비스용)은 더 정교하지만, 핵심 골격만 generic하게 추렸다. 자기 레포 경로에 맞게 고쳐 써라.
> 동작 개요는 → [[01-learn-compounding-kb]]

---

세션 끝 또는 한 작업 완료 후 실행한다. 이번 세션의 **재사용 가능한 교훈만** 영구 KB로 적립한다.

## Phase 1: Merit Gate (가장 먼저 — 대부분 0개 저장이 정상)

지식의 단위는 "작업"이 아니라 재사용 가능한 **문제/실수/결정**이다.
"구현한 것"을 적지 말고 "다음에 또 만날 함정·패턴"만 적는다. 아래를 모두 만족할 때만 통과:

- 6개월 뒤 *다른* 일을 하던 내가/팀원이 이걸로 실제 시간을 아끼는가?
- 다른 합리적인 사람이 다르게 정할 수 있는 **비자명한** 선택/실수인가?
- 이번 세션이 *기존 패턴의 단순 적용*일 뿐이면 → 신규 항목 **0개**가 정상.

게이트를 통과한 것만 아래로 분류한다. 통과한 게 없으면 "Nothing to save." 출력 후 종료.

## Phase 2: 중복·모순 점검

영구 KB를 먼저 grep 한다.

```bash
grep -rni "[핵심 키워드]" docs/wiki/gotchas.md docs/wiki/decisions.md docs/adr/
```

- **높은 중복**: 기존 항목을 **업데이트** (중복 추가 금지)
- **부분 중복**: 관련 항목 근처에 새로 추가
- **모순**: 기존 항목을 직접 **고치거나 대체** (⚠️ 마커만 달고 방치 금지)

## Phase 3: 자리 정하기 (분기표는 routing-table.md)

| 교훈 유형 | 자리 |
|---|---|
| 코드/작업 안티패턴 (틀림→맞음) | `docs/wiki/gotchas.md` 표 1줄 |
| 화면·기능 스코프 결정 (왜 이 동작) | `docs/wiki/decisions.md` 항목 |
| 아키텍처 결정 (ADR 기준 3개+ 해당) | `docs/adr/ADR-NNN-*.md` 초안 |

## Phase 4: 컴파일 규칙

1. 자리를 먼저 읽는다 — 유사 항목 있으면 추가가 아닌 **업데이트**.
2. 참조는 **실제 코드 경로**로. 코드 사본을 통째로 넣지 않는다(금방 낡는다).
3. 컴파일 조건(모두 충족): 아직 안 쓰인 코드에도 적용되는 미래지향 규칙 / 특정 파일 1곳에 묶이지 않음 / 다른 일을 하는 사람도 적용 가능.

## Phase 5: 승격 검토 (선택)

같은 실수가 **2회+ 재발**했거나 *어떤 미래 파일에서도* 어기면 안 되는 하드 제약이면
→ `.claude/rules/*.md`(상시 로드)로 승격 **제안**. 단, 상시 로드는 매 세션 토큰을 먹으니 **사람 확인 후**에만.

## Phase 6: 요약 출력

```
Compiled to: [gotchas 1줄 / decisions 항목 / ADR-NNN  — 또는 "Nothing to save"]
```
