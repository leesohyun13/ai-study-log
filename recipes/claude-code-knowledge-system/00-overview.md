> 실제 운영 중인 안드로이드 앱 프로젝트에서 쓰는 셋업이고, 예시는 전부 generic(TodoApp 등)으로 바꿔서 올린다.

## 🧭 30초 설명 (개발 몰라도 됨)

- **무엇:** AI랑 일하며 배운 걸 한곳에 쌓아, 다음에 또 안 까먹게 하는 **습관 + 자동화 셋업**.
- **왜 필요:** 안 하면 같은 실수를 매번 다시 밟고, "지난번에 왜 이렇게 정했지"를 매번 다시 논쟁한다. 배운 게 그때그때 휘발된다.
- **누구용:** AI(ChatGPT·Claude·Cursor 등)로 일하는 사람 **누구나. 개발자가 아니어도 된다.**

> 🔵 **개발자는** 2번(배선표)과 2차 카드(02~04)까지.

## 0. 이건 카드 한 장이 아니라 "세트"다

조각마다 따로 떼어 쓸 수 있게 카드를 나눴다. **전부 따라 할 필요 없다. 필요한 것만 가져가라.**

| 카드 | 한 줄 | 대상 |
|---|---|---|
| **00-overview** (이 문서) | 전체 지도 + 뭘 가져갈지 | 전원, 먼저 읽기 |
| **[[01-learn-compounding-kb]]** | 일한 *후* 배운 걸 복리로 적립 | **누구나** (AI로 일하면 됨) |
| [[02-brainstorm-gate]] | 코드 짜기 *전* 멈추고 되묻기 | 🔵 개발자 |
| [[03-spec-at-a-glance]] | 상태/동작을 표로 한눈에 | 🔵 개발자·기획 |
| [[04-review-gates]] | 구현 후 3단 검증 | 🔵 개발자 |

→ **읽는 순서**: 누구나 `00`(이 문서) → `01`. 개발자는 이어서 `02 → 03 → 04`. 비개발자는 `01`까지면 충분하다.

## 1. 시스템은 두 개의 반쪽이다

| 반쪽 | 언제 작동 | 구성 | 대상 |
|---|---|---|---|
| **A. 일한 *후* — 지식 적립** | 작업이 끝나고 | `/learn`, Merit Gate, `gotchas`/`decisions`/`ADR`, `knowledge-search` | **누구나** |
| **B. 일하기 *전·중* — 실행 통제** | 코드 짜기 전~검증 | Brainstorm Gate → 계획 → 리뷰 게이트 → 스펙 작성 | 주로 개발자 |

이 세트는 **A(적립)** 를 `01`에서, **B(실행 통제)** 를 `02~04`에서 다룬다. 비개발자는 A만 가져가도 충분하다.

## 2. ⭐가장 중요 — Claude에 "어떻게 로드되는가" (배선표)

> 🔵 **여기부터는 개발자용.** 비개발자는 3번 차용 사다리로 건너뛰어도 된다.

**복붙해도 안 먹히는 #1 이유:** 파일마다 Claude가 읽어들이는 방식이 다르다. 같은 `.md`라도 어디 두느냐로 동작이 갈린다.

| 위치 | 로딩 방식 | 용도 |
|---|---|---|
| `CLAUDE.md` (프로젝트 루트) | **매 세션 자동** | 상시 규칙. `@경로`로 다른 파일 포함 가능 |
| `.claude/rules/*.md` | **항상 로드** | 절대 어기면 안 되는 제약 (= 매 세션 토큰 비용 발생) |
| `.claude/commands/*.md` | `/이름` 호출 시 | 수동 트리거 명령 (예: `/learn`) |
| `.claude/skills/*/SKILL.md` | 키워드 자동 발동 | 설명문에 맞는 상황이 오면 자동 |
| `.claude/agents/*.md` | 서브에이전트 위임 시 | 위임 작업 정의 |
| 그냥 `.md` (예: `docs/`, `workflows/`) | 다른 문서가 "읽어라" 참조할 때만 | 온디맨드 절차서·지식 |

**핵심 트레이드오프:** "항상 로드"(rules)는 편하지만 **매 세션 토큰을 먹는다.** 그래서 대부분은 온디맨드(검색될 때만 읽힘)로 두고, *정말 매번 강제할 것만* rules로 올린다. 이 비용 감각이 없으면 컨텍스트가 금방 비대해진다.

## 3. 차용 사다리 — 전부 하지 말고 1칸부터

1. **`gotchas.md` 한 장** 만들기 — 누구나, AI 없이도 됨. "또 틀린 것"을 `틀림 → 맞음` 1줄씩.
2. **`/learn` 한 줄 명령** 붙이기 — 세션 끝에 적을 것만 자동 적립.
3. **작업 전 `knowledge-search`** — 시작 전 과거 교훈을 먼저 끌어오기.
4. (개발자) **02~04 파이프라인**까지 — 2차 배치 참고.

대부분은 1~2번만으로도 충분히 복리가 돈다.
