# 🍳 AI가 엉뚱한 방향으로 달려가기 전에 멈춰 세우기 (Brainstorm Gate)

- **작성자 / 직무:** 안현호 / 개발자(안드로이드)
- **날짜:** 2026-06-17
- **상태:** ✅ 성공 (실제 운영 중)

> 🔵 **이 카드는 개발자용**이지만, **원리는 비개발 업무에도 그대로 통한다**(6번 참고).
> 전체 지도 → [[00-overview]] · 짝이 되는 적립 루프 → [[01-learn-compounding-kb]]

---

**1. 문제 (Problem)**
> AI는 모호한 요청에도 **확신을 갖고** 바로 코드를 쏟아낸다.
> - 요청이 두루뭉술한데도 일단 출발 → 절반쯤 가서야 "아, 이게 아닌데" → 다 뒤엎는다.
> - 원인은 AI가 아니라 **모호함을 안은 채 출발한 것**이다. 시작점에서 안 풀린 모호함은 결과물에서 폭발한다.

**2. 원래 걸리던 시간**
> 잘못 짠 걸 갈아엎고 다시 짜는 재작업. "다 만들었는데 방향이 틀려서 처음부터" 가 제일 비싸다.

**3. 쓴 도구 · 프롬프트 (Recipe)**

### 원리 (누구나)

큰일을 시키기 전, **실행 전에 한 번 멈추는 관문**을 둔다.

1. **의도 구분** — "바꿔라"가 명확할 때만 바로 실행. *조사/평가/질문* 의도거나 모호하면 → 멈추고 합의부터.
2. **먼저 탐색** — 관련 자료(기존 코드/과거 결정/[[01-learn-compounding-kb]]의 KB)를 먼저 훑어 영향 범위를 판단.
3. **모호한 점을 한 번에 객관식으로 질문** — A/B/C 보기 + **추천안과 이유**를 붙여서. 질문을 작업 중간중간 흘리지 말고 *시작 전에 모아서* 던진다.
4. **범위·파일 목록 확정 → 계획 승인 후 시작.**

> ⭐ 핵심은 **"추천안 없는 빈 질문을 던지지 않는 것"**. "어떻게 할까요?"만 물으면 부담을 전가할 뿐이다. "A를 추천합니다(이유). B/C도 가능"이어야 상대가 1초 만에 고른다.

### 언제 관문을 켜나 (트리거)

아무 일에나 켜면 잔소리꾼이 된다. 아래일 때만:
- 바꿀 대상이 **2곳 이상**이거나 범위를 **모를 때**
- **공용 인터페이스·공유 데이터**를 건드릴 때 (다른 곳에 영향)
- 막연한 동사만 있을 때 ("개선해줘", "정리해줘" + 구체 대상 없음)
- 한 문장인데 **결과물이 안 그려질 때**

→ 사소한 1줄 수정은 관문 없이 그냥 한다.

### 구현 (Claude Code 전제)

- 이 관문을 절차서(`.md`)로 만들어 `CLAUDE.md`에서 "이 조건이면 이 절차서를 읽어라"로 연결.
- 로딩은 **온디맨드** — 트리거 조건일 때만 읽힘(상시 로드 아님, 토큰 절약). 배선표 → [[00-overview]].

**4. 줄어든 시간 / 효과**
> - "절반 가서 뒤엎기"가 확 줄었다.
> - 질문이 *앞쪽에 모이니* 작업 중 끊기는 일이 줄고, 합의된 계획대로 직진.

**5. 함정 (안 되던 것 / 주의점)** ← 제일 중요
> - **모든 사소한 일에도 관문을 켰다가** 답답해서 안 쓰게 됨. → 트리거 기준(2곳+/공유 변경 등)을 명확히. 작은 건 그냥 한다.
> - **추천안 없는 빈 질문.** 사용자가 결정 부담을 다 떠안는다. → 항상 추천+이유.
> - **작업 중 새 의문이 생겼는데 멋대로 가정하고 진행.** → 멈추고 다시 묻는다(임의 추측 금지).
> - **"질문 → 승인"을 건너뛰고 바로 계획을 실행.** → 승인 전엔 계획까지만.

**6. 재사용 가능성**
> - **비개발 동일**: AI에게 큰 산출물(기획서·카피 세트·분석)을 시키기 전, "객관식으로 먼저 합의" 한 단계만 넣어도 재작업이 급감한다.
> - **도구 무관**: ChatGPT·제미나이 등 어디서나. "바로 시키지 말고, 모호하면 보기 만들어 합의부터."
