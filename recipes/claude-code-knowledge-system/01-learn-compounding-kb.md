# 🍳 AI랑 일하며 배운 걸 "복리로" 쌓는 셋업 (/learn + 영구 KB)

- **작성자 / 직무:** 안현호 / 개발자(안드로이드)
- **날짜:** 2026-06-17
- **상태:** ✅ 성공 (실제 운영 중)

> 전제: 이 카드의 **원리**는 어떤 AI 도구에서도 통한다(ChatGPT·Cursor·제미나이·노션 메모도 OK). **구현 예시**는 Claude Code 기준이다. 둘을 구분해서 적었다.
> 전체 지도와 배선표는 → [[00-overview]]

---

**1. 문제 (Problem)**
> AI랑 작업하면 *그 세션 안에선* 똑똑하다. 그런데 세션이 끝나면 배운 게 휘발된다.
> - 다음 세션의 AI는 지난번 함정을 모른 채 **같은 실수**를 반복한다.
> - "지난번에 이거 왜 이렇게 정했더라"가 사라져서 **같은 결정을 다시 논쟁**한다.
> - 팀원/미래의 내가 합류해도 맥락이 머릿속에만 있어 **전수가 안 된다.**

**2. 원래 걸리던 시간**
> 정량화는 어렵다. 매 작업마다 "과거에 밟은 함정 다시 밟기 + 정한 걸 다시 정하기"로 새는 시간이다. 비개발로 치면 "지난 캠페인에서 안 먹혔던 걸 까먹고 또 하기"와 같은 종류의 손실.

**3. 쓴 도구 · 프롬프트 (Recipe)**

### 먼저, 실제로 이렇게 돌아간다 (구체 예시)

> 용어 없이 한 사이클만 따라가 보자.
>
> - **월요일** — 작업하다 "목록 패딩을 컨테이너에 주면 클릭이 안 먹힌다"를 발견하고 고침.
> - **작업 끝** — `/learn` 한 줄 입력. AI가 *"이건 다음에 또 만날 함정이네"* 라고 판단 → 메모 1줄 적립: **목록 패딩은 항목 단위로.**
> - **다음 주, 다른 화면 작업** — 시작 전 AI가 그 메모를 자동으로 읽고 *"패딩은 항목 단위로 주세요"* 라고 먼저 알려줌. → **같은 실수를 두 번 안 함.**
>
> 이게 전부다. **"배운 걸 그때그때 적고, 다음에 자동으로 꺼내 쓴다."** 아래는 이걸 *망치지 않고* 운영하는 규칙이다.

### 원리 (누구나) — 세 동작의 루프

1. **작업 끝 → 적립**: "이번에 *구현한 것*"이 아니라 **"다음에 또 만날 *함정·결정*"만** 한 문서에 적는다.
2. **작업 시작 → 회수**: 새 작업 전, 그 문서를 먼저 검색해 관련 교훈을 끌어온다.
3. **문지기(Merit Gate, = "적을 가치가 있나" 심사)**: 아무거나 적으면 KB가 쓰레기장이 된다. 기준은 단 하나 —
   > *"6개월 뒤 **다른** 일을 하던 내가/팀원이 이걸로 실제 시간을 아끼는가?"* → 아니면 안 적는다.
   >
   > **대부분의 세션은 0개 저장이 정상이다.** (한 작업 결과물을 그대로 쌓지 않는다)

### 적는 곳을 성격별로 3단 분리 (이게 핵심)

> 용어 풀이: **gotchas** = "또 틀린 것" 메모 · **decisions** = "왜 이렇게 정했나" 메모 · **ADR** = 되돌리기 힘든 큰 결정 기록(대안+이유 포함)

| 성격 | 자리 | 예시 (generic) |
|---|---|---|
| 코드/작업 안티패턴 (틀림→맞음) | `gotchas.md` 표 1줄 | "목록 패딩을 컨테이너에 주면 클릭영역이 깨짐 → 항목 단위로 줘라" |
| 한 화면·기능 범위의 가벼운 결정 | `decisions.md` 1줄 | "빈 목록일 때 토스트 대신 일러스트 표시 — 이유: 첫 진입 이탈률" |
| 되돌리기 힘든 구조 결정 | `ADR-NNN.md` (대안+이유) | "상태관리 라이브러리 X 채택 — 대안 Y/Z 검토, 선택 이유 …" |

가벼운 결정과 무거운 결정을 섞지 않는 게 포인트다. 무거운 건 "대안과 선택 이유"가 본문이고(=ADR), 가벼운 건 한 줄이면 된다(=decisions).

### 구현 (Claude Code 전제)

- **`/learn`** : 세션 끝에 부르는 한 줄 명령. 위 Merit Gate를 돌려서, 적을 게 있으면 적합한 자리에 자동 컴파일.
- **`knowledge-search`** : 작업 시작 전 KB를 뒤져 관련 교훈을 먼저 가져오는 서브에이전트.
- **`solutions/`** : **임시** 초안 폴더. 상세 초안을 잠깐 썼다가 → 영구 자리로 컴파일 → **삭제**한다. (영구 저장소가 아님 — 5번 함정 참고)
- 명령·표 전문은 → `starter-kit/`에 복붙용으로 넣어뒀다.

### 워크플로우 한 눈에

```
작업 시작 → knowledge-search (과거 교훈 회수)
   → 작업 …
작업 끝   → /learn → Merit Gate 통과한 것만 → gotchas / decisions / ADR
```

**4. 줄어든 시간 / 효과**
> - 같은 함정 재발이 눈에 띄게 줆 (새 세션의 AI가 KB를 컨텍스트로 받으니까).
> - 새 사람/새 AI 세션 온보딩이 "이 문서들 읽어" 한 줄로 끝난다.
> - 솔직히 정량 수치는 못 댄다. 핵심은 "안 잃어버린다"는 것이고, 그게 복리의 본질이다 — 1년 쌓이면 차이가 난다.

**5. 함정 (안 되던 것 / 주의점)** ← 이 칸이 제일 중요하다
> 전부 실제로 데어본 것들이다.
> - **임시 폴더를 영구 저장소로 썼다가 망함.** 초안을 안 지우니 중복·낡은 문서가 쌓여 검색이 오염됐다. → "컴파일 후 삭제"로 전환. 영구 자리엔 요약 + **실제 코드 경로**만 남긴다.
> - **다 적으려는 충동.** 작업로그를 그대로 쌓으면 KB가 일기장이 된다. → Merit Gate로 막는다. **안 적는 게 기본값.**
> - **코드 사본을 문서에 통째로 붙이기.** 코드는 바뀌는데 문서는 안 바뀌어서 금방 거짓말이 된다. → 실제 코드 경로만 링크(원본 코드가 진실의 출처).
> - **"항상 로드"의 토큰 비용.** 자주 어기는 규칙을 상시 로드 자리로 올리면 편하지만 **매 세션 토큰을 먹는다.** → 정말 매번 강제할 것만 승격(사람이 확인 후). 배선표는 [[00-overview]].
> - **모순 누적.** 새 교훈이 옛 항목과 충돌할 때 ⚠️ 마커만 달고 두면 둘 다 살아남아 더 헷갈린다. → 옛 항목을 직접 고치거나 대체한다.

**6. 재사용 가능성**
> - **도구 무관**: ChatGPT·Cursor·제미나이·심지어 노션 메모로도 원리 그대로. AI를 안 써도 된다.
> - **직무 무관**: 마케터 = "캠페인 함정/결정 로그", 기획자 = "기능 결정 로그(왜 이 동작을 택했나)". `gotchas`+`decisions` 두 장이면 시작.
> - **시작점**: `starter-kit/` 복붙 → `gotchas.md` 한 줄부터. 단계별로 늘리는 법은 [[00-overview]]의 "차용 사다리".
